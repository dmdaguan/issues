What is hasura/issues?

hasura/issues is a simple view that pulls data from github and gitlab repos that helps the entire team understand project progress. It also has a few utilities to automatically send out emails, show blocking tasks and extract time spent on a task (using issue description).

Why this tool?

The idea is to remove all overhead with task tracking and project progress. No extra tools, no multiple sources of truth about what is going on for developers, project managers etc.

Features

  1. Project progress
  2. List issues for the same milestone across github, gitlab repos
  3. Parse issue description to show blocking tasks
  4. Automatic progress progress emails for an active milestone
  5. Automatic daily report of issues opened and closed in case of no active milestone

Configuration

hasura/issues is a node service that is deployed per project. All configuration is only via env variables. Here's a list of env vars, what they mean and how to configure them:

  1. GITLAB: XXXX_YOUR_GITLAB_API_TOKENXXXX
  2. GITHUB: XXXX_YOUR_GITHUB_API_TOKENXXXX
  3. MILESTONES: {"milestone_x": {"start": "YYYY-MM-DD", "end": "YYYY-MM-DD"}, "milestone_y"...}
